Write articles by Hexo

Blog

Write-articles-by-Hexo
用hexo写文章攻略，做一个记录。

创建文章

Blog目录右键 GIT BASH HERE

Hexo title "[article name]"

[article name]:文章名
文件名：[article name].md
资源目录：source/_post

文章编写 Markdown

工具：markdown

有道云笔记编写
Typora : [https://www.typora.io/](https://www.typora.io/)

外链资源使用

—config.yml: post_asset_folder 选项设置成 true

npm install hexo-asset-image --save 安装插件

source/_post 新建文件夹：与文章名同名

使用方式：![示例图片]([article name]/example.PNG "示例图片")

模板

模板地址：scaffolds/post.md

1.头部设置

title: Title
date: YYYY-MM-DD HH:MM:SS
tags: [tag1, tag2, ...]
categories: category

注意，属性和属性值之间必须有一个空格，否则会解析错误

2.首页设置

在Hexo框架博客的首页会显示文章的内容（默认显示文章的全部内容），如果当文章太长的时候就会显得十分冗余，所以我们有必要对其进行精简，只需在文章中使用 <!--more--> 标志，表示只会显示标志前面的内容

部署发布

hexo clean
hexo g -s：测试
hexo d：发布

Markdown语法

1.标题

# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题

2.粗体、斜体、删除线和下划线

*斜体*
**粗体**
***加粗斜体***
~~删除线~~

3.超链接

[链接文字](链接地址 "链接描述")
例如：[我的博客](https://unrealsage.github.io/ "我的博客")

<链接地址>
例如：<https://forwhfang.github.io>

4.引用

> 文字引用

5.代码

`单行代码`

6.分割线

7.列表

8.表格

9.图片

![图片文字] (图片地址 "图片描述")
例如：![示例图片] (https://www.example.com/example.PNG "示例图片")
